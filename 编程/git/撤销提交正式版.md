1. 先是只修改了工作区的流程

要在工作区修改后将更改暂存，先 pull 最新的更改，然后应用暂存的更改，你可以按以下步骤操作：

```
git add .
git stash//将暂存的更改存储为一个临时的变更集（Stash）
git pull
git stash pop


```

1. 撤销工作区的更改（未提交的更改）
 ```
git checkout -- <file>

git checkout -- .

```
2.撤销暂存区的更改，已经add, 但是没有提交

```
git reset HEAD <file>
git reset HEAD .




```
3.撤销最近的一次提交（还没有推送到远程仓库）
```
git reset --soft HEAD^

git reset --hard HEAD^


```
4. 撤销已经推送到远程仓库的提交
```
git revert <commit>
git push origin <branch>
或者强制

git reset --hard <commit>
git push origin <branch> --force



```
